# Решение проблемы CORS с Hugging Face API

## Проблема
При открытии `index.html` через `file://` протокол браузер блокирует запросы к внешним API из-за CORS политики.

## Решения

### Вариант 1: Локальный веб-сервер (Рекомендуется)

Запустите локальный сервер вместо открытия файла напрямую:

**Python:**
```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

**Node.js:**
```bash
npx http-server -p 8000
```

**PHP:**
```bash
php -S localhost:8000
```

Затем откройте: `http://localhost:8000`

### Вариант 2: Использовать CORS прокси

Можно использовать публичные CORS прокси, но это не рекомендуется для production.

### Вариант 3: Использовать другой API

- Google Vision API (требует ключ, но работает с CORS)
- Использовать локальную модель (TensorFlow.js)

### Вариант 4: Отключить CORS в браузере (Только для разработки!)

⚠️ **Небезопасно!** Только для тестирования:

**Chrome:**
```bash
chrome.exe --user-data-dir="C:/Chrome dev session" --disable-web-security
```

**Firefox:**
Установите расширение CORS Everywhere

## Рекомендация

Используйте **Вариант 1** - локальный веб-сервер. Это самый простой и безопасный способ.
